GoogleはGemini CLIを廃止して、Antigravity CLI（以下Agy）に移行予定です。AgyはGemini CLIと同様に、コマンドライン型のAIエージェントハーネスです。

私はGemini CLIのユーザーでしたので、Agyへ移行進める中、ステータスラインの表示だけはできていませんでした。

https://antigravity.google/docs/cli-statusline

ステータスラインは最近のAIエージェントには搭載されている機能で、各種情報を画面下部へ表示する機能です。
Agyのステータスラインは、シェルスクリプト等（おそらくスクリプトが動作するならなんでも）を使って表示内容をカスタマイズ可能です。

ただ、私の環境のWindowsでは何の言語で作っても情報が何も返ってこない状況でした。

最近AgyのリポジトリーのIssueに解決方法が上がっていて、やっと動作させることができたので今回はその方法を紹介します。
とは言え、Issueに書いてある内容を実行するだけなので、非常に簡単です。

https://github.com/google-antigravity/antigravity-cli/issues/207#issuecomment-4691158364

## 手順

1. Git for Windowsをインストールする。
https://git-scm.com/download/win

2. インストール後、Windowsのユーザー環境変数 `PATH` の最下部へGitのインストールディレクトリー（e.g., C:\Program Files\Git\bin）を追加する。

3. PATH変更後PowerShellを起動して、`sh --version` コマンドが動作することを確認する。

    sh.exeがPowerShell上で動作するのでこの設定以後は、シェルスクリプトを簡単に動作させることが可能になります。

4. Agyのステータスラインを表示するためのシェルスクリプトを作成する。

    私が動作を確認したスクリプトはGistで公開しています。
    
    https://gist.github.com/naoyukik/04d7e5c4cefa3922763a9bc5b945abe5

5. ステータスライン表示用のスクリプトを `/statusline` コマンドで設定する。

    設定はAgy起動後に以下のようにコマンドを引数で渡します。

    ```bash
    /statusline sh C:\path\to\your\statusline.sh
    ```

## スクリプト解説
ほとんどが公式のexampleを流用しているだけですが、動作させるにあたっていくつか動かない点を修正したのと、いくつか表示項目を変更していますので簡単に解説します。

私の方ではnormalize_num関数を追加してあります。名前は数字の正規化ですが、実際には"\r"の削除もまとめてやっています。これがないとステータスラインが正しく表示されないことがありました。
数字に限らずデータは取れてるのに上手く表示されない場合、"\r"が混入していないか確認すると良いです。
どんなデータがAgyから出力されているかは、スクリプト中に `INPUT` 変数にデータを取得してログに出力できるようにコメントアウトしてあるので、必要に応じて確認可能です。

なお取得可能なデータについては、先述の公式ドキュメントの "Available JSON fields" セクションをご確認ください。

詳細な表示項目については、各種変数とoutputの内容をご確認ください。

## まとめ
以上、AgyのステータスラインをWindowsで表示する方法でした。

Git for Windowsをインストールして、sh.exeが動作するように環境変数を設定するだけで、あとはシェルスクリプトを用意するだけで簡単に表示できるようになります。

副産物として、sh.exeで動作するスクリプトがPowerShell上で動かせるようになるので、Agy以外の用途でも便利に使えるようになりそうです。
